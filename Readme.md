## 复习一下左连接

你想复习 MySQL 中左连接（LEFT JOIN） 的核心含义和用法，我会从定义、通俗解释、
示例（结合你的图书管理系统场景）、关键点这几个维度，帮你把这个知识点梳理清楚。
1. 左连接的核心含义
   左连接（LEFT JOIN / LEFT OUTER JOIN，两者等价，OUTER 可省略）是 
   MySQL 中多表关联查询的一种方式，核心规则是：以左表为基准，返回左表的所有记录，
   同时匹配右表中满足关联条件的记录；如果右表没有匹配的记录，则右表的字段值显示为 NULL。

2. 通俗比喻
   可以把左连接理解为：“我（左表）要把我的所有数据都列出来，
   然后找找你（右表）有没有和我匹配的，有就一起显示，没有的话你的位置就空着（NULL）”。

## @Results({
            @Result(column = "id",property = "id")
    }) column对应数据库 property对应类？

在 MyBatis 的 @Results/@Result 注解中：
column：必填，指的是数据库查询结果集中的列名（可以是表的原始字段名，也可以是 SQL 查询时的别名）；
property：必填，指的是Java 实体类（POJO/Entity）中的属性名；
两者的作用：建立 “数据库列” 和 “实体类属性” 的映射关系，MyBatis 会把查询结果中 column 对应的值，赋值给实体类中 property 对应的属性。
